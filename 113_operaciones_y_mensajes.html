1.1.3 Operaciones y mensajes

Operaciones

Las operaciones son acciones o comportamientos que un objeto puede realizar. En el contexto de la programación orientada a objetos se conocen como “métodos o funciones”.

Representan acciones de un objeto

Las operaciones representan las acciones que un objeto puede realizar en base a los mensajes que recibe de otros objetos o del entorno.

Incluyen diferentes actividades

En dichas acciones se pueden incluir cálculos, modificaciones de estado del objeto, interacciones con otros objetos, y diferentes tipos de actividades que el objeto pueda realizar.

Importancia

  • Las operaciones son fundamentales porque encapsulan el comportamiento de los objetos y definen cómo interactúan dentro de un sistema.

  • Identificar y definir las operaciones adecuadas para cada objeto es esencial para diseñar un sistema coherente y eficiente.

Ejemplo

Un cajero automático puede realizar las siguientes operaciones:

  • depositar()
  • retirar()
  • consultarSaldo()
  • transferirDinero()

Mensajes

Los mensajes son el medio por el cual los objetos se pueden comunicar entre si.

  • Veamos a un mensaje como una solicitud enviada desde un objeto (remitente) a otro objeto (receptor).


Ahora bien, las clases deben de trabajar juntas para lograr los objetivos del diseño:

  • Para esto los mensajes hacen que el objeto que lo recibe realice una acción específica.
  • Esta acción ocurre cuando una operación es ejecutada.

La interacción entre los objetos se puede observar en el siguiente esquema:

1. Una operación en el ObjetoEmisor crea un mensaje con el siguiente formato mensaje(<parámetros>).
2. Los parámetros indican que el ObjetoReceptor es el objeto que debe ser activado por el mensaje.
3. La operación en ObjetoReceptor implica recibir el mensaje y los datos necesarios para que la acción se realice correctamente.

Ejemplo

Un ejemplo común de ver es el de una televisión y su control remoto:

  • Cuando desea ver algún programa de televisión, busca el control remoto y presiona el botón de encendido para ver su programa favorito.

Por lo cual, ocurre el siguiente proceso:

  1. El control remoto le envía un mensaje a la televisión para que se encienda.
  2. La televisión recibe el mensaje, lo identifica como una petición para que se encienda y lo hace.

De igual manera, el control remoto puede enviar otros mensaje con peticiones como cambiar de canal, ajustar el volumen, activar los subtítulos.
